langchain框架学习-1

2023-11-21 9 min read

LangChain 是一个开发由语言模型驱动的应用程序的框架。我们相信最强大和不同的应用程序不仅会通过 API 调用语言模型， 还会：数据感知 : 将语言模型连接到其他数据源；具有代理性质 : 允许语言模型与其环境交互。

介绍

LangChain是一个用于开发由语言模型驱动的应用程序的框架。
它使应用程序能够：

  • 上下文感知：将语言模型连接到上下文源（提示指令、少量镜头示例、内容以使其响应为基础等）
  • Reason：依靠语言模型进行推理（关于如何根据提供的上下文回答、采取什么行动等）

该框架由几个部分组成。

  • LangChain 库：Python 和 JavaScript 库。包含无数组件的接口和集成，将这些组件组合成链和代理的基本运行时，以及链和代理的现成实现。
  • LangChain模板：一组易于部署的参考架构，适用于各种任务。
  • LangServe：用于将LangChain链部署为REST API的库。
  • LangSmith：一个开发者平台，可让您调试、测试、评估和监控基于任何LLM框架构建的链，并与LangChain无缝集成。

安装

pip install langchain

QuickStart

LangChain提供了许多模块，可用于构建语言模型应用程序。 模块可以在简单的应用程序中用作独立模块，并且可以针对更复杂的用例进行组合。 组合由LangChain表达式语言（LCEL）提供支持，它定义了一个统一的接口，许多模块实现了该接口，从而可以无缝地链接组件。

最简单和最常见的链包含三件事：

  • LLM/聊天模型：语言模型是这里的核心推理引擎。为了使用LangChain，您需要了解不同类型的语言模型以及如何使用它们。
  • 提示模板：这为语言模型提供说明。这控制了语言模型的输出内容，因此了解如何构造提示和不同的提示策略至关重要。
  • 输出分析器：这些分析器将来自语言模型的原始响应转换为更可行的格式，从而便于在下游使用输出。

LLM/聊天模型

有两种类型的语言模型：

  • LLM： 底层模型将字符串作为输入并返回字符串
  • ChatModel： 基础模型将消息列表作为输入，并返回消息。

可以当做LLM模型直接收字符串,聊天模型只接收消息列表

消息必须具备的两个属性: 内容和实体

langchain提供的几个对象来区分不同的角色:

  • HumanMessage 用户角色
  • AIMessage: AI助手
  • SystemMessage: 系统
  • FunctionMessage,ToolMessage, 使用函数工具返回的输出.

此外,也可以自定义角色.

不同类型的模型及对应的输入

from langchain.llms import OpenAI
from langchain.chat_models import ChatOpenAI

llm = OpenAI() # 需要配置参数进行初始化
chat_model = ChatOpenAI()

from langchain.schema import HumanMessage

text = "What would be a good company name for a company that makes colorful socks?"
messages = [HumanMessage(content=text)]
llm.invoke(text)
# >> Feetful of Fun
chat_model.invoke(messages)
# >> AIMessage(content="Socks O'Color")

提示模板

大多数的LLM应用程序不会将用户输入直接传递到LLM中,通常是将用户输入添加到更大的文本中,该文本提供了有关这个特定任务的其他上下文. (俗称提示词).

prompttemplate可以将用户的输入转换为格式化的输入，并和预制的逻辑绑定到一起输入。

from langchain.prompts import PromptTemplate

prompt = PromptTemplate.from_template("What is a good name for a company that makes {product}?")
prompt.format(product="colorful socks")

#What is a good name for a company that makes colorful socks?

应用2

from langchain.prompts.chat import ChatPromptTemplate

template = "You are a helpful assistant that translates {input_language} to {output_language}."
human_template = "{text}"

chat_prompt = ChatPromptTemplate.from_messages([
    ("system", template),
    ("human", human_template),
])

chat_prompt.format_messages(input_language="English", output_language="French", text="I love programming.")

[
    SystemMessage(content="You are a helpful assistant that translates English to French.", additional_kwargs={}),
    HumanMessage(content="I love programming.")
]

输出解析器:
OutputParsers将语言模型的原始输出转换为可下游使用的格式。 主要有几种类型，包括：

  • 将文本转换为结构化信息（例如 JSON）LLM
  • 将 a 转换为字符串ChatMessage
  • 将调用中返回的额外信息（如 OpenAI 函数调用）转换为字符串。

自定义的一个Demo：

from langchain.schema import BaseOutputParser

class CommaSeparatedListOutputParser(BaseOutputParser):
    """Parse the output of an LLM call to a comma-separated list."""


    def parse(self, text: str):
        """Parse the output of an LLM call."""
        return text.strip().split(", ")

CommaSeparatedListOutputParser().parse("hi, bye")
# >> ['hi', 'bye']

链

我们可以把这些逻辑组合成一个链，用于完成一个复杂的场景：

from typing import List

from langchain.chat_models import ChatOpenAI
from langchain.prompts import ChatPromptTemplate
from langchain.schema import BaseOutputParser

class CommaSeparatedListOutputParser(BaseOutputParser[List[str]]):
    """Parse the output of an LLM call to a comma-separated list."""


    def parse(self, text: str) -> List[str]:
        """Parse the output of an LLM call."""
        return text.strip().split(", ")

template = """You are a helpful assistant who generates comma separated lists.
A user will pass in a category, and you should generate 5 objects in that category in a comma separated list.
ONLY return a comma separated list, and nothing more."""
human_template = "{text}"

chat_prompt = ChatPromptTemplate.from_messages([
    ("system", template),
    ("human", human_template),
])
chain = chat_prompt | ChatOpenAI() | CommaSeparatedListOutputParser()
chain.invoke({"text": "colors"})
# >> ['red', 'blue', 'green', 'yellow', 'orange']

关于将其转换为接口API并应用，这里不再追加。

安全问题

在构建此类应用程序时，开发人员应记住遵循良好的安全实践：

  • 限制权限：根据应用程序的需要专门限定权限范围。授予宽泛或过多的权限可能会引入重大安全漏洞。为避免此类漏洞，请考虑使用只读凭据、禁止访问敏感资源、使用沙盒技术（例如在容器内运行）等，以适合您的应用程序。
  • 预测潜在的滥用：正如人类可能会犯错一样，大型语言模型 （LLM） 也会犯错。始终假定任何系统访问或凭据都可以以分配的权限允许的任何方式使用。例如，如果一对数据库凭据允许删除数据，则最安全的假设是任何能够使用这些凭据的 LLM 实际上可能会删除数据。
  • 纵深防御：没有任何安全技术是完美的。微调和良好的链设计可以减少但不能消除大型语言模型 （LLM） 出错的几率。最好结合多层安全方法，而不是依赖任何单一的防御层来确保安全性。例如：同时使用只读权限和沙盒来确保 LLM 只能访问明确供它们使用的数据。

不这样做的风险包括但不限于：

  • 数据损坏或丢失。
  • 未经授权访问机密信息。
  • 关键资源的性能或可用性受到影响。

具有缓解策略的示例方案：

  • 用户可以要求有权访问文件系统的代理删除不应删除的文件或读取包含敏感信息的文件的内容。若要缓解，请将代理限制为仅使用特定目录，并且仅允许其读取或写入可安全读取或写入的文件。请考虑通过在容器中运行代理来进一步沙盒化代理。
  • 用户可以要求对外部 API 具有写入权限的代理将恶意数据写入 API，或从该 API 中删除数据。若要缓解，请为代理提供只读 API 密钥，或将其限制为仅使用已可抵抗此类滥用的终结点。
  • 用户可以要求有权访问数据库的代理删除表或更改架构。若要缓解，请将凭据范围限定为代理需要访问的表，并考虑颁发只读凭据。

如果要构建访问外部资源（如文件系统、API）的应用程序 或数据库，请考虑与公司的安全团队交谈，以确定如何最好地 设计和保护您的应用程序。
